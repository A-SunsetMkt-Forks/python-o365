FluentMessage¶

class O365.fluent_message.FluentMessage(message=None, json_data=None, verify=True)[source]¶

Bases: object

Makes a wrapper for message object to make fluent api calls for sending

Parameters:
  • message – Message object to wrap in fluent api
  • json_data – Takes json if you have a pre-existing message to create from. this is mostly used inside the library for when new messages are downloaded.
  • verify – whether or not to verify SSL certificate
bcc(*recipients)[source]¶

Set the BCC recipients list.

Each recipient can be either of the below:

type:

dict - this must to be a dictionary formatted as such

{
    "EmailAddress": {
        "Address":"user@domain.com"
    }
}

with other options such as “Name” with address. but at minimum it must have this.
type:

str - simple email address in form of “user@domain.com”
type:

Contact - Contact object (Contact is part of this library)
type:

Group - Group object, which is a list of contacts (Group is part of this library)
Parameters:recipients – List of recipients
Returns:copy of this object
Return type:FluentMessage

Note

This method is part of fluent api and can be chained

send(user_id=None, **kwargs)[source]¶

Send the email

Parameters:user_id – User id (email) if sending as other user
Returns:copy of this object
Return type:FluentMessage

Note

This method is part of fluent api and can be chained

Note

The success/failure of this action can be obtained from success and error_message attributes after executing this function

Example:
my_obj.one().two().finish()
if not my_obj.is_success:
    print(my_obj.error_message)

this will return success/failure of finish action
